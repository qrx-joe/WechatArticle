## 主帖

同一个项目我做了两遍，一次以 Codex 为主力，一次以 Claude Code 为主力。三个月后坐下来一看——功能重叠 90%，得到了同一个产品的两个草稿。

第一版（Codex）代码粗糙但工程齐全：CI、hooks、回滚机制都有。第二版（Claude Code）规范完美但缺了水电——没有 CI，没有 hooks，连回滚都没了。

最讽刺的发现：不是两个 AI 决定了代码风格，是它们放大了我潜意识里已有的偏好。第一次我想"先做出来看看"，第二次我想"这次一定做好"。AI 只是一面镜子。

最终我把两版优点合并到新仓库。教训很简单：迭代比重写更勇敢，够好比完美更有价值。

两个仓库都开源了，欢迎围观我的重复建设现场。

## 评论1

有人问我"哪个 AI 编程更强"——这个问题本身就是陷阱。Codex 倾向"先完成再完美"，工程直觉强；Claude Code 倾向"先定义再实现"，代码更整洁但容易陷入规范先行的陷阱。两个都很强，但它们会把你推向不同的极端。

## 评论2

最大的浪费不是代码，是完美主义。回头看，我不需要开第二个仓库，只需要把第一版的 CI 修好、类型补全、重构那个丑陋的函数就够了。"重来一次会更好"是最大的幻觉。
