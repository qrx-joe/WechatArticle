## 二、排版操作步骤

### 2.1 选择排版工具

推荐使用以下工具之一（选一个顺手的）：

| 工具 | 特点 | 网址 |
|------|------|------|
| **壹伴** | 功能最全，支持数据监控 | 浏览器插件 |
| **秀米 XIUMI** | 模板丰富，拖拽方便 | xiumi.us |
| **135编辑器** | 免费功能多 | 135editor.com |
| **Markdown Here** | 极简，适合纯文字 | 浏览器插件 |

**推荐：秀米 或 壹伴**（支持 Markdown 导入，省去手动调格式）

### 2.2 导入文章

**方案 A：使用秀米（推荐）**

1. 打开秀米官网，登录
2. 点击「新建一个图文」
3. 点击编辑器左上角「...」→「导入 Word / Markdown」
4. 选择 `article.md` 文件
5. 等待导入完成，检查格式

**方案 B：使用壹伴**

1. 安装壹伴浏览器插件
2. 进入公众号后台「新的创作」→「图文消息」
3. 点击壹伴工具栏的「导入」按钮
4. 粘贴 Markdown 内容或导入文件

**方案 C：纯手动（最稳妥）**

1. 进入公众号后台 mp.weixin.qq.com
2. 「新的创作」→「图文消息」
3. 逐段复制 `article.md` 内容粘贴到编辑器

### 2.3 格式调整

导入后，按以下标准调整：

| 元素 | 设置 |
|------|------|
| 正文字号 | 15px（手机阅读舒适） |
| 正文颜色 | #333333（深灰，比纯黑柔和） |
| 行间距 | 1.75 倍 |
| 段落间距 | 段前/段后各空一行 |
| 小标题字号 | 17px，加粗 |
| 小标题颜色 | #000000 或品牌色 |
| 引用块 | 灰色背景 #F5F5F5，左边 3px 橙色竖线 |
| 强调文字 | 加粗即可，不要用彩色 |

### 2.4 特殊格式处理

**引言引用块（第 3 行）：**

```
> 不是我开始得早，是我差点就放弃了。
```

- 设置为「引用」样式
- 背景：#F5F5F5
- 左边框：3px solid #FF6B35（亮橙色）
- 文字居中或左对齐均可

**"三天。"单独强调（第 49 行）：**

- 单独成行
- 字号：18px 或 20px
- 加粗
- 居中
- 上下各空两行

**列表（第 19-23 行）：**

- 保持无序列表样式
- 列表项缩进一致

### 3.2 图片设置

每张图片插入后：

1. **尺寸**：宽度 100%（占满屏幕），高度自适应
2. **边距**：图片上下各空一行
3. **小字说明**（可选）：图片下方加一行 12px 灰色小字
   - 例：「那个项目在 GitHub 上睡了八个月」
4. **图片质量**：确保上传原图，不要压缩

### 3.3 封面图特殊处理

1. 在公众号后台编辑器底部，找到「封面」区域
2. 点击「从正文选择」或「上传图片」
3. 选择 `01-cover.png`
4. **关键**：点击「封面编辑」调整图片显示区域
   - 确保核心元素（女生+电脑+光）在正方形裁切范围内
   - 公众号列表中显示的是正方形缩略图

## 四、发布前检查

### 4.1 手机预览（必须做）

1. 点击编辑器底部「预览」按钮
2. 输入自己的微信号，发送预览
3. **在手机上完整阅读一遍**，检查：
   - [ ] 段落是否太长（超过 5 行要拆分）
   - [ ] 图片是否变形或模糊
   - [ ] 小标题是否醒目
   - [ ] "三天。"是否足够突出
   - [ ] 链接是否能点击（如有）
   - [ ] 整体阅读节奏是否流畅

### 4.2 常见错误检查

| 检查项 | 问题描述 | 修复方法 |
|--------|----------|----------|
| 错别字 | AI/人工校对遗漏 | 通读一遍，重点关注技术名词 |
| 格式丢失 | Markdown 导入后样式错乱 | 手动重新设置标题、加粗、引用 |
| 图片过大 | 超过 5MB 无法上传 | 用 tinypng.com 压缩 |
| 图片过小 | 小于 200px 显示模糊 | 重新生成更高分辨率版本 |
| 封面裁切 | 列表模式下显示不全 | 调整封面编辑的裁切区域 |
| 段落过长 | 手机阅读体验差 | 超过 5 行拆成两段 |

### 4.3 摘要设置

在公众号后台编辑器底部，找到「摘要」区域：

**推荐摘要：**

```
一个经济学女生，用AI三天完成了拖了八个月的项目。这个故事不是关于AI有多强，是关于一个人如何被AI改变了"敢想"的勇气。
```

- 字数：54 字（在 120 字限制内）
- 作用：朋友圈分享时显示的简介

## 五、发布操作

### 5.1 发布流程

1. 确认所有检查项已完成
2. 点击编辑器底部「保存」
3. 点击「保存并群发」或「发布」（见下方区别）
4. 扫码确认管理员身份
5. 等待审核通过（通常几分钟到几小时）

### 5.2 「群发」vs「发布」的区别

| 方式 | 效果 | 适用场景 |
|------|------|----------|
| **群发** | 推送给所有关注者，有消息提醒 | 常规推文，希望获得阅读量 |
| **发布** | 不推送，只生成文章链接 | 测试、占位、或不想打扰用户 |

**建议：第一篇用「群发」**，既然是开号宣言，应该让关注者看到。

### 5.3 发布后操作

1. **转发到朋友圈**（开号第一篇，自己转发很重要）
2. **转发到相关社群**（如有）
3. **观察数据**：24 小时后查看阅读量、点赞、在看、留言
4. **回复留言**：前 10 条留言尽量都回复，增加互动

## 六、备选方案

### 6.1 如果没有配图

如果暂时来不及生成配图，可以：

1. **纯文字发布**：公众号文章完全可以纯文字，配一张封面图即可
2. **用简单色块代替**：秀米/135编辑器有纯色块组件，用不同颜色区分章节
3. **用 emoji 代替**：标题前加 emoji 增加视觉层次

### 6.2 如果排版工具打不开

直接用公众号自带编辑器：

1. mp.weixin.qq.com
2. 新建图文消息
3. 手动粘贴文字
4. 用编辑器的「引用」「列表」「加粗」功能调格式
5. 上传图片插入

自带编辑器功能足够，只是没有第三方工具方便。

## 八、紧急联系

如果发布过程中遇到问题：

| 问题 | 解决方案 |
|------|----------|
| 公众号后台打不开 | 清除浏览器缓存，或换 Chrome/Edge |

| 格式导入后错乱 | 手动逐段调整，或用「纯文本粘贴」再重新设置格式 |
